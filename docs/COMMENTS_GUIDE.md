# CAX 项目代码注释规范

## 📝 注释标准

本项目采用详细的中文注释规范，确保代码易于理解和维护。

## 📋 注释类型

### 1. 文件头注释

每个源文件开头必须包含文件头注释，说明文件用途：

```v
// ============================================================================
// CAX - Comandiux for V Language
// 命令行系统工具 V 语言实现
// ============================================================================
// 功能：实现原始 cax.exe 的全部 255 个命令
// 作者：CAX Team
// 版本：0.0.1
// 许可证：MIT
// ============================================================================
```

### 2. 函数注释

每个函数前必须有详细注释：

```v
// ============================================================================
// 函数名称
// ============================================================================
// 功能：简要描述函数功能
// 参数：
//   param1 - 参数 1 说明
//   param2 - 参数 2 说明
// 返回值：
//   返回值说明
// ============================================================================

fn function_name(param1 type, param2 type) return_type {
    // 函数实现
}
```

### 3. 块注释

用于分隔代码块，使用分隔线：

```v
// ============================================================================
// 功能模块名称
// ============================================================================

// 功能说明
code_line_1()

// 另一个功能
code_line_2()
```

### 4. 行内注释

用于解释复杂逻辑：

```v
// 检查是否为反斜杠（转义字符开始）
if text[i] == `\\` && i + 1 < text.len {
    // 根据下一个字符确定转义类型
    match text[i + 1] {
        `n` { 
            // 换行符 (Line Feed)
            result << u8(`\n`)
            i += 2 
        }
    }
}
```

## 🎯 注释内容要求

### 必须包含的信息

1. **功能说明** - 这段代码做什么
2. **参数说明** - 每个参数的用途和类型
3. **返回值** - 返回什么结果
4. **关键算法** - 复杂逻辑的解释
5. **边界条件** - 特殊情况的处理

### 推荐做法

✅ **好的注释：**
```v
// 计算实际秒数（参数单位是秒/10）
timeout_sec := timeout.int() / 10
```

❌ **避免的注释：**
```v
// 除以 10
timeout_sec := timeout.int() / 10
```

## 📖 示例

### 完整函数注释示例

```v
// ============================================================================
// 转义字符解析函数
// ============================================================================
// 功能：将字符串中的转义序列转换为实际字符
// 支持的转义字符：
//   \n - 换行 (LF)
//   \t - 制表符 (TAB)
//   \r - 回车 (CR)
//   \0 - 空字符 (NULL)
//   \b - 退格 (Backspace)
//   \e - 转义 (Escape)
//   \s - 空格 (Space)
//   \\ - 反斜杠
//   \" - 双引号
// 参数：
//   text - 包含转义序列的原始字符串
// 返回值：
//   解析后的字符串
// ============================================================================

pub fn parse_escape_sequences(text string) string {
    // 创建字节数组存储结果
    mut result := []u8{cap: text.len}
    mut i := 0
    
    // 遍历输入字符串
    for i < text.len {
        // 检查是否为反斜杠（转义字符开始）
        if text[i] == `\\` && i + 1 < text.len {
            // 根据下一个字符确定转义类型
            match text[i + 1] {
                `n` { 
                    // 换行符 (Line Feed)
                    result << u8(`\n`)
                    i += 2 
                }
                // ... 其他转义类型
            }
        } else {
            // 普通字符直接添加
            result << text[i]
            i++
        }
    }
    
    // 转换为字符串返回
    return result.bytestr()
}
```
